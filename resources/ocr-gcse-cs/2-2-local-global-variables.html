🎯 Why Does It Matter Where You Create a Variable?

When programs grow beyond a few lines, we organise code into subroutines (functions and procedures). This creates a question: where can each variable be accessed from?

This concept is called scope, and understanding it prevents bugs that can be incredibly frustrating to track down.

🏠 Think of it like this...

Imagine a house with multiple rooms. Some items belong in one specific room (your toothbrush stays in the bathroom). Other items are shared throughout the house (the WiFi password works everywhere). Variables work the same way in programs.

What You'll Learn

By the end of this resource, you'll be able to look at any variable in a program and immediately know:

  • Where it was created (its declaration point)
  • Where it can be used (its scope)
  • Why the programmer chose that approach

🔍 What is Scope?

Definition Scope

The region of a program where a variable can be accessed. A variable only exists within its scope.

When you create a variable, it doesn't automatically become available everywhere in your program. The location where you declare it determines where it can be used.

Visualising Scope
🌍 Main Program
gameTitle = "Space Quest" ← visible everywhere
📦 calculateScore()
bonus = 50 ← only visible here
📦 displayMenu()
menuChoice = 1 ← only visible here

Notice how gameTitle sits outside the functions — it's accessible from anywhere. But bonus and menuChoice are trapped inside their own functions.

⚠️ Key Point

If you try to use a variable outside its scope, you'll get an error. The program won't recognise the variable name because it "doesn't exist" in that context.

📍 Local Variables

Definition Local Variable

A variable declared inside a subroutine. It can only be accessed within that subroutine and is destroyed when the subroutine ends.

Local variables are contained — they exist only for as long as their function is running. Once the function finishes, the variable is deleted from memory.

def calculateDiscount(price): discountRate = 0.15 # Local variable savings = price * discountRate # Local variable return savings result = calculateDiscount(100) print(result) # Works: 15.0 print(discountRate) # ERROR! discountRate doesn't exist here

In this example, discountRate and savings are local to the calculateDiscount function. They're created when the function runs and destroyed when it ends.

Why Use Local Variables?

💾 Memory efficient — only exists while needed, then automatically cleaned up
🔒 Protected — can't be accidentally changed by other parts of the program
🐛 Easier debugging — if something's wrong, you know exactly where to look
♻️ Reusable — subroutines can be copied to other programs without conflicts
💡 Best Practice

Use local variables as your default choice. Only use global variables when you have a specific reason to share data across the entire program.

🌍 Global Variables

Definition Global Variable

A variable declared outside all subroutines, usually at the start of a program. It can be accessed from anywhere in the program.

Global variables are like shared resources — every part of your program can read and (with care) modify them.

playerLives = 3 # Global variable highScore = 0 # Global variable def loseLife(): global playerLives # Tell Python we mean the global one playerLives = playerLives - 1 def checkGameOver(): if playerLives == 0: # Can read global without 'global' keyword print("Game Over!") loseLife() print(playerLives) # Output: 2

Notice the global keyword in Python. When you want to modify a global variable inside a function, you must declare your intention with this keyword.

Why Use Global Variables?

🔗 Shared access — multiple subroutines can use the same data
📝 Single declaration — define once, use everywhere
🔧 Configuration values — perfect for settings that apply program-wide
📊 Program state — useful for tracking things like scores, lives, or login status
⚠️ Warning: Use Sparingly

Global variables can make programs harder to debug because any part of the code could change them. When something goes wrong, you have to search the entire program to find the problem. Use them only when genuinely needed.

Let's put everything together and see how these two types of variables compare:

📍

🌍
Global Variable
Lives in the main program. Shared, persistent, and accessible everywhere.

🎒 Backpack vs Locker Analogy

Local variables are like items in your backpack during a class — you bring what you need, use it, then pack it away when class ends. Global variables are like items in your school locker — available all day, any class can access them, but anyone could also mess with them.

📝 Decision Guide

Use Local When...

  • The variable is only needed in one function
  • It's a temporary value or calculation
  • You want to protect it from being changed elsewhere

Use Global When...

  • Multiple functions need the same data
  • It's a program-wide setting or state
  • The value must persist between function calls
