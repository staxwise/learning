QUESTION 1

CPU Architecture

9 marks
Class performanceMixed — Q1B and Q1D were the sticking points
1(a) — 2 marks · DESCRIBE
Describe the function of cache memory.
What to write — 2 marks
Cache memory stores frequently or recently used data/instructions MARK 1 so that the CPU can access them quickly — faster than accessing RAM MARK 2.
⚠ Common mistakes
✗"Memory that helps the CPU" — too vague, 0 marks. Must mention what it stores AND why it's faster.
✗Confusing cache with RAM — cache is smaller, faster, and sits closer to the CPU.
1(b) — 2 marks · EXPLAIN
Explain how cache memory can affect the performance of the CPU.
Class avg ~62% — many students made the link but didn't complete the chain
What to write — 2 marks (chain must be complete)
A larger cache can store more frequently used data MARK 1, which reduces the need to access the slower RAM/main memory, allowing the CPU to process instructions more quickly / reducing latency MARK 2.

The chain: More/larger cache → less RAM access → faster CPU performance. Break the chain and you lose marks.
1(c) — 1 mark · STATE
State the purpose of the Arithmetic Logic Unit (ALU).
What to write — 1 mark
Performs arithmetic calculations (e.g. addition, subtraction) and/or logical comparisons (e.g. AND, OR, NOT). MARK 1
💡 Note

"Processes data" alone = 0. You must mention arithmetic OR logical. Most of the class got this — well done.

1(d) — 4 marks · DESCRIBE
Describe how the fetch–execute cycle handles instructions within the CPU.
Class avg ~74% — most scored 2–3/4. Getting all 4 requires covering all three stages.
Model answer — 4 marks available across FETCH, DECODE, EXECUTE
FETCH: The Program Counter (PC) holds the address of the next instruction. MARK The address is copied to the MAR and the instruction is fetched from RAM into the MDR. MARK It is then copied to the CIR and the PC is incremented. MARK

DECODE: The Control Unit decodes the instruction in the CIR. MARK

EXECUTE: The instruction is executed by the relevant component (e.g. ALU). Results may be stored back in memory/registers. MARK
📌 The Registers to Know

Registers

  • PC — Program Counter: holds address of next instruction
  • MAR — Memory Address Register: holds address being accessed
  • MDR — Memory Data Register: holds data fetched from memory
  • CIR — Current Instruction Register: holds instruction being decoded/executed

Components

  • CU — Control Unit: decodes instructions, coordinates CPU
  • ALU — Arithmetic Logic Unit: carries out calculations
  • PC increments after each fetch (points to next instruction)
  • Cycle repeats continuously while powered
QUESTION 2

Storage & Data Representation

12 marks
🔴 High Priority — Class lost marks here

Q2A and Q2B were both weaker than expected. Q2B (the calculation) should be 3 reliable marks every exam — fix the method and lock them in.

2(a) — 6 marks · LEVEL RESPONSE · COMPARE
A gamer wants to store high-resolution video games. Compare magnetic storage and solid-state storage. You might want to refer to: Cost, Speed, Durability.
Class avg ~78% — most scoring 3–4/6. Top band (5–6) requires all three aspects WITH conclusions.
Level 3: 5–6 marks
Covers all three aspects (cost, speed, durability) with clear development. Links to the gaming context. Reaches a conclusion on which is better and why. Uses technical terminology throughout.
Level 2: 3–4 marks
Covers at least two aspects. Some development, some links to gaming. Generally accurate terminology.
Level 1: 1–2 marks
Only one aspect, or very superficial coverage. Limited terminology.
Model answer — Level 3 (5–6 marks)
Cost: Magnetic hard drives (HDDs) are cheaper per gigabyte than SSDs, meaning a gamer could store more games for the same budget on an HDD. However, SSD prices have fallen significantly and they are now affordable for most consumers. COST ✓

Speed: SSDs have faster read/write speeds with no moving parts, meaning games load significantly faster — often in seconds compared to minutes on an HDD, which has seek time due to spinning platters. For a gaming context, this is a major advantage. SPEED ✓

Durability: SSDs have no moving parts and are therefore more shock-resistant and durable. HDDs can fail if moved or dropped during operation. This matters less for a desktop gaming PC but is significant for portable gaming. DURABILITY ✓

Conclusion: For a gaming use case, SSD is recommended for its speed and reliability, though an HDD remains a valid choice for budget-conscious bulk storage of many games. CONCLUSION ✓
💡 The Level 3 Formula

To reach 5–6: (1) cover all three bullet points + (2) develop each with gaming context + (3) write a conclusion. Missing the conclusion is the most common reason students fall to Level 2.

2(b) — 3 marks · CALCULATE
An image has a resolution of 100 × 100 pixels and a colour depth of 4 bits per pixel. Calculate the file size in kilobytes (KB). Show your working.
🔴 Class avg ~55% — the most fixable question on the paper. One formula, three steps.
STEP 1
100 × 100 = 10,000 pixels  →  MARK 1
STEP 2
10,000 × 4 = 40,000 bits  →  MARK 2
STEP 3
40,000 ÷ 8 = 5,000 bytes  →  5,000 ÷ 1,000 = 5 KB  →  MARK 3
⚠ Where students went wrong
✗Forgetting to ÷8 to convert bits to bytes — the most common single error
✗Treating 100×100 as the answer (forgetting to multiply by colour depth)
✗Method marks are awarded even if arithmetic is wrong — always show your working
📌 The Formula (memorise this)

File size (bits) = Width × Height × Colour depth  →  ÷8 for bytes  →  ÷1,000 for KB  →  ÷1,000 for MB

2(c) — 3 marks · EXPLAIN
Explain why lossy compression is suitable for a video file on a website but not suitable for a text file.
What to write — need at least 1 point for each file type
For video (suitable): Video files are very large and lossy compression achieves significant size reduction. MARK The data removed (such as imperceptible colour differences between frames) is not noticeable to a viewer, so quality loss is acceptable. MARK

For text (not suitable): Every character in a text file is essential for meaning — losing even one character would corrupt the document or change its meaning entirely. MARK

Networks & Protocols

9 marks

Mixed performance across this question — Q3B was strong (87%) but Q3A and Q3C had more yellow than expected.

3(a) — 2 marks · IDENTIFY
Identify the protocol used for: (1) Sending an email    (2) Securely accessing a bank website
Exact answers required (AO questions)
Sending an email: SMTP (sending) or POP3 / IMAP (receiving) — accept any MARK 1
Securely accessing a bank website: HTTPS (only — not HTTP) MARK 2

For the bank website: HTTPS only. "SSL" or "TLS" alone = 0. SSL/TLS are protocols within HTTPS, not the answer to this question. If you wrote HTTP you scored 0 — the S matters.

3(b) — 2 marks · EXPLAIN
Explain the concept of layers in network protocols.
What to write — 2 marks
Network communication is divided into separate layers, each with a specific function/responsibility. MARK 1 Each layer communicates only with the layers immediately above and below it, working together to enable complete data transmission. MARK 2
3(c) — 1 mark · GIVE
Give one benefit of using a layered approach.
Class avg ~55% — surprisingly weak for 1 mark. Must be specific.
Any ONE of these — 1 mark
• Easier to update or modify one layer without affecting the others OR
• Allows different manufacturers to develop different layers independently OR
• Easier to troubleshoot — can identify which layer has a problem OR
• Allows standardisation across different systems/devices
⚠ Common mistake

"Makes it faster" — 0 marks without explanation. You must name a specific benefit, not a vague claim.

3(d) — 4 marks · EXPLAIN
Explain two advantages of using cloud storage for files.
Class avg ~70% — most scored 2/4. Getting 4 requires identifying the advantage AND explaining it.
2 marks per advantage (identify + explain)
Advantage 1: Files can be accessed from anywhere / any device with an internet connection. MARK 1 This means users can work remotely or from different locations without needing their original device. MARK 2

Advantage 2: Cloud storage providers automatically back up data (redundancy). MARK 3 This protects against data loss if the user's local device fails, is damaged, or stolen. MARK 4
💡 The 2-mark pattern

Each advantage = identification (1 mark) + explanation/development (1 mark). "You can access from anywhere" alone = 1 mark, not 2. Always add the why this is useful sentence.

Security

8 marks
4(a) — 2 marks · DEFINE
Define phishing.
Must have BOTH components — 2 marks
A fraudulent attempt to obtain sensitive information (passwords, bank details, personal data) MARK 1 by disguising as a trustworthy entity in electronic communication (e.g. fake emails appearing to be from a bank). MARK 2
⚠ Must have BOTH

4(b) — 1 mark · STATE
State one way a user could identify a phishing email.
Any ONE — 1 mark (class scored well here ✅)
• Suspicious/misspelled sender email address OR Poor spelling/grammar OR Urgent/threatening language OR Requests for personal/financial info OR Suspicious links OR Generic greeting ("Dear Customer") OR Unexpected attachments
4(c) — 3 marks · EXPLAIN
Explain how user access levels could help prevent a data breach within an organisation.
What to write — 3 from these points, must link to breach prevention
  • Different users are given different permissions/rights, so they can only access data relevant to their role. MARK
  • This limits the damage if one account is compromised — an attacker can only access that user's permitted data, not the entire system. MARK
  • Sensitive data is only accessible to authorised personnel, reducing the risk of accidental or intentional exposure. MARK
  • Creates accountability — actions can be traced to specific users, deterring misuse. MARK
4(d) — 2 marks · DESCRIBE
Describe the purpose of penetration testing.
🔴 Class avg ~58% — "testing security" or "testing the network" = 0. Need the simulate/identify structure.
What to write — 2 marks
Penetration testing involves simulating a cyber attack on a system (an authorised/ethical hacking attempt) MARK 1 in order to identify vulnerabilities before real attackers do, allowing the organisation to fix security flaws proactively. MARK 2
⚠ Why students lost marks

"Testing the network" = 0. "Testing security" = 0. You must include: (1) the simulate/ethical hacking aspect AND (2) the purpose — finding vulnerabilities before real attackers. Think of it as: attack your own system on purpose, so you can fix what you find.

Systems Software — Operating System

4 marks
⚠ Class avg ~68% — both roles must be answered

Most students answered one role well but gave a weaker answer on the second. Both memory management AND multitasking must be addressed for full marks.

5(a) — 4 marks · DESCRIBE (2+2)
Describe the role of the operating system in: (1) memory management    (2) multitasking.
Model answer — max 2 marks per role
Memory Management (2 marks — any 2):
• Allocates memory to programs/processes when they are launched. MARK
• Deallocates / frees memory when programs close. MARK
• Manages virtual memory when RAM becomes full. MARK
• Prevents programs from accessing each other's memory space. MARK

Multitasking (2 marks — any 2):
• Allows multiple programs to appear to run simultaneously by switching between them rapidly (context switching). MARK
• Allocates CPU time to each process (time slicing / scheduling). MARK
• Prioritises tasks so more important processes receive more CPU time. MARK
📌 Key vocab that gets marks

Memory management: allocate, deallocate, virtual memory, address space. Multitasking: time slicing, context switching, process scheduling, concurrent. Using these terms signals understanding to the examiner.

QUESTION 6 🔴 PRIORITY

Ethical, Legal & Cultural Issues

10 marks
🔴 Biggest single mark-loss on the paper

Q6A (8 marks) had a class average of ~60% — approximately 3.2 marks per student left on the table. This is a level response question with four bands. Most students scored Level 2 (3–4) or low Level 3 (5). Scoring Level 4 (7–8) requires all three issue types covered, developed with examples, and balanced perspectives.

6(a) — 8 marks · LEVEL RESPONSE · DISCUSS
A global social media company uses algorithms to filter content for users in different countries and collects large amounts of personal data. Discuss the cultural, privacy, and legal issues raised.
Level 4: 7–8 marks — what it takes
Covers all three areas (cultural + privacy + legal) in depth. Well-developed arguments with specific examples. Shows understanding of tensions and conflicts between different interests. Balanced — considers multiple perspectives (e.g. benefits vs harms, individual vs state, company vs user).
Level 3: 5–6 marks
Covers all three areas with some depth. Points developed with some examples. Some attempt at balance or different viewpoints. Where most of your class was — to move up, add specific examples and balance.
Level 2: 3–4 marks — where many students were
Covers some but not all three areas. Some development but lacks specific examples. Limited consideration of different perspectives.
Level 1: 1–2 marks
Limited, superficial. Only addresses one area. Little development.
Model content — what a Level 4 answer covers
CULTURAL ISSUES: Algorithm-driven content filtering may reflect the cultural values of the company's home country, suppressing content from minority cultures or languages. This raises concerns about cultural imperialism — where one country's norms are imposed globally. Filter bubbles limit users' exposure to diverse viewpoints, reinforcing existing beliefs rather than challenging them. Different countries have genuinely different standards for acceptable content (e.g. political speech, religious content), creating genuine ethical dilemmas about whose values should govern global platforms.

PRIVACY ISSUES: The company collects large volumes of personal data, often without users fully understanding what is collected or how it is used. Data may be sold to third parties for targeted advertising without truly informed consent. Users have little transparency over what data is held or how profiles are built. Data breaches could expose sensitive personal information. The concept of the right to be forgotten is relevant — can users have their data removed?

LEGAL ISSUES: GDPR (General Data Protection Regulation) requires companies to have a legal basis for processing personal data, to gain consent, and to allow users to access and delete their data. Different countries have different laws, creating compliance challenges for a global platform. The Computer Misuse Act may apply if data is accessed without authorisation. There are tensions between freedom of speech laws (stronger in some countries) and censorship regulations in others — the company must navigate these competing legal frameworks.

BALANCE: On the other hand, content filtering can protect users from harmful material (hate speech, misinformation, illegal content). Personalisation driven by data collection genuinely improves user experience. Completely unfiltered platforms raise their own serious harms. There is no simple right answer — the key is appropriate regulation and transparency.
💡 The Level 4 Checklist — annotate your answer now

Did your answer include?

  • At least 2 points on cultural issues?
  • At least 2 points on privacy issues?
  • At least 2 points on legal issues?
  • Specific examples (GDPR, right to be forgotten, filter bubbles)?

For Level 4 specifically

  • Did you consider multiple perspectives (not just negatives)?
  • Did you discuss tensions/conflicts between issues?
  • Did you use a range of technical vocabulary?
  • Was your answer organised (cultural → privacy → legal)?

RAM vs ROM. Write R for RAM, O for ROM. (1) Is volatile memory    (2) Stores boot instructions

RAM: volatile (data lost when powered off), stores currently running programs and data, fast, can be read and written. ROM: non-volatile (data permanent), stores BIOS/boot instructions, cannot normally be written to.

10(a) — 2 marks · BENEFIT or DRAWBACK
Star vs Mesh topology. Tick whether each statement is a benefit or drawback.

Answers — both relate to STAR topology

📌 Star topology: Key points

Benefits

  • Easy to add/remove devices
  • Fault in one device doesn't affect others
  • Easy to troubleshoot
  • High performance (dedicated connection)

Drawbacks

  • Central switch/hub = single point of failure
  • More cable needed than bus topology
  • Dependent on central device
10(b) — 2 marks · SWITCH or ROUTER

Answers (class scored 97% here ✅)
Sends data between different networks → Router ✓ MARK 1

📌 Switch vs Router in one sentence each

Switch: connects devices within a LAN, learns MAC addresses to direct frames only to the intended recipient. Router: connects different networks (e.g. LAN to WAN/internet), routes packets using IP addresses.

QUESTION 11

Systems Software & Legislation

3 marks

Q11A was mostly fine. Q11B (proprietary licence) had lower performance (~50%) — make sure you know which tick is which.

11(a) — 2 marks · FILL IN THE BLANKS
Defragmentation improves file access speed by moving file fragments, so they are stored _____ on the disk, reducing the _____ time of the read/write head.
Answers
…stored contiguously / together / sequentially / next to each other MARK 1 …reducing the seek / movement / travel time MARK 2
📌 Why defragmentation helps

Over time, files get split into fragments stored in different locations on an HDD. The read/write head has to physically move (seek) to each fragment location — this takes time. Defragmentation reorganises fragments so they are stored contiguously (all in one place), reducing seek time and improving speed. Note: SSDs don't need defragmenting — no moving parts means seek time isn't an issue.

11(b) — 1 mark · TICK one benefit AND one drawback
Proprietary licence — tick one benefit and one drawback.
Class avg ~50% — the wrong options are easy to confuse with open source characteristics
Correct answers (AO — exact ticks required)
Benefit: ✓ Protects source code MARK
Drawback: ✓ Cannot access or modify source code
⚠ The traps in this question
✗"Users can edit and improve software freely" — this is an open source feature, NOT a benefit of proprietary
✗"Software can be redistributed without restrictions" — again open source, not proprietary
✗"Guaranteed free updates" — 0 (not a drawback of proprietary; often proprietary software does provide updates)
📌 Proprietary vs Open Source — quick reference

Proprietary

  • Source code is hidden/protected
  • Cannot be freely modified or redistributed
  • Usually costs money (licence fee)
  • Developer controls updates and support
  • Examples: Windows, Microsoft Office, Photoshop

Open Source

  • Source code is freely available
  • Can be modified and redistributed
  • Usually free to use
  • Community-driven development
  • Examples: Linux, LibreOffice, Firefox
