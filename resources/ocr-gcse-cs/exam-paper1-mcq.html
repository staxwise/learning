1.1 Systems Architecture

Cache Memory Spec 1.1.1

Cache Memory
A small amount of very fast memory located on or near the CPU that stores frequently used data and instructions.
Key Facts
  • Purpose: Reduces the number of times the CPU needs to fetch data from slower RAM
  • Speed hierarchy: Registers (fastest) → Cache → RAM → Secondary Storage (slowest)
  • Levels: L1 (smallest, fastest, on CPU) → L2 → L3 (largest, slowest)
  • More cache = faster processing (fewer slow RAM fetches needed)
💡 Common Exam Question

"Why don't computers just use cache instead of RAM?" — Answer: Cache is extremely expensive to manufacture. Having 16GB of cache would cost thousands of pounds.

CPU Components Spec 1.1.1

ALU (Arithmetic Logic Unit)
Performs all arithmetic calculations (+, −, ×, ÷) and logical operations (AND, OR, NOT, comparisons like > and =).
CU (Control Unit)
The "brain" of the CPU. Decodes instructions, sends control signals, and coordinates data movement between components.

Key Registers

Register Full Name Purpose
MAR Memory Address Register Holds the address of the memory location to read/write
MDR Memory Data Register Holds the data being transferred to/from memory
PC Program Counter Holds the address of the next instruction to fetch
ACC Accumulator Stores results of ALU calculations

Fetch-Execute Cycle Spec 1.1.1

FETCH
Get instruction from memory
→
DECODE
CU interprets instruction
→
EXECUTE
Carry out the instruction
Detailed Steps
  • Fetch: PC sends address to MAR → Memory sends data to MDR → PC increments (+1)
  • Decode: CU reads the instruction and works out what operation is needed
  • Execute: The instruction is carried out (e.g., ALU performs calculation)

1.2 Memory and Storage

Storage Technologies Spec 1.2.2

Type Magnetic (HDD) Solid State (SSD) Optical (DVD/Blu-ray)
Speed Slow (moving parts) Very fast (no moving parts) Slow
Capacity High (up to 20TB) Medium-High Low (4.7GB–50GB)
Cost per GB Low (cheapest) Higher Very low
Durability Fragile (moving parts can break) Durable (no moving parts) Can scratch
Noise Audible (spinning) Silent Audible when spinning
Best for Servers, bulk storage Laptops, gaming, OS drives Distributing media
💡 Scenario Questions

Recommend HDD when: Large capacity needed, budget is limited (e.g., backup server)
Recommend SSD when: Speed matters, device is portable (e.g., laptop), durability needed

Compression Spec 1.2.3

Aspect Lossy Compression Lossless Compression
Data lost? Yes – permanently removed No – all data retained
Quality Reduced (may be noticeable) Perfect (identical to original)
File size reduction Large reduction Smaller reduction
Examples JPEG, MP3, streaming video PNG, ZIP, FLAC
Best for Web images, music streaming Text files, code, medical images
Why Compress Files?
  • Faster transmission: Smaller files transfer quicker over networks
  • Less storage space: More files can fit on a drive
  • Lower bandwidth costs: Especially important for streaming services

Data Calculations Spec 1.2.3

Image File Size
Width (pixels) × Height (pixels) × Colour Depth (bits per pixel)
Sound File Size
Sample Rate (Hz) × Duration (seconds) × Bit Depth × Channels
Unit Conversions (OCR Standard)
  • 8 bits = 1 byte
  • 1000 bytes = 1 KB (kilobyte)
  • 1000 KB = 1 MB (megabyte)
  • 1000 MB = 1 GB (gigabyte)
💡 Calculation Example

Image: 800 × 600 pixels, 24-bit colour depth
= 800 × 600 × 24 = 11,520,000 bits
= 11,520,000 ÷ 8 = 1,440,000 bytes
= 1,440,000 ÷ 1000 = 1,440 KB = 1.44 MB

1.3 Networks, Connections & Protocols

Key Protocols Spec 1.3.2

Protocol Purpose Key Detail
SMTP Sending emails Simple Mail Transfer Protocol
IMAP Receiving emails Syncs with server – emails stay on server
POP Receiving emails Downloads and deletes from server
HTTP Web browsing Insecure – data not encrypted
HTTPS Secure web browsing Encrypted – uses SSL/TLS certificates
FTP Transferring files File Transfer Protocol – upload/download
💡 IMAP vs POP

IMAP: Better for accessing email on multiple devices (phone + laptop) – stays synced
POP: Better when you only use one device or have limited server storage

IP Addressing Spec 1.3.2

Aspect IPv4 IPv6
Format 4 groups of decimal numbers 8 groups of hexadecimal
Example 192.168.1.1 2001:0db8:85a3:0000:0000:8a2e:0370:7334
Addresses available ~4.3 billion ~340 undecillion (practically unlimited)
Why IPv6? We ran out of IPv4 addresses due to the growth of internet-connected devices

Concept of Layers Spec 1.3.2

Layered Architecture
Dividing the complex task of networking into smaller, manageable layers where each layer has a specific job (e.g., the TCP/IP model).
Benefits of Layers
  • Self-contained: Changes in one layer (e.g., new WiFi standard) don't break others
  • Interoperability: Hardware/software from different manufacturers can work together
  • Easier troubleshooting: Problems can be isolated to a specific layer
  • Specialisation: Developers can focus on one layer without understanding all others

The Cloud Spec 1.3.1

Cloud Computing
Using remote servers accessed via the internet to store, manage, and process data instead of local servers or personal computers.
Advantages Disadvantages
Access files from anywhere with internet Requires internet connection
Automatic backups Security managed by third party (trust issues)
Easy collaboration with others Ongoing subscription costs
Scalable – increase storage easily Privacy concerns – who can access your data?

1.4 Network Security

Phishing Spec 1.4

Phishing
A social engineering attack where criminals send emails or messages pretending to be from a reputable source (bank, HMRC, etc.) to trick people into revealing personal information.
How to Identify Phishing
  • Poor spelling/grammar: Legitimate companies proofread communications
  • Urgent/threatening language: "Your account will be closed in 24 hours!"
  • Suspicious links: Hover over links to see the real URL
  • Unexpected attachments: Don't open files you weren't expecting
  • Generic greetings: "Dear Customer" instead of your actual name
💡 Prevention Methods

User training: Educate staff to recognise phishing attempts
Spam filters: Automatically block suspicious emails before they reach users
Two-factor authentication: Even if credentials are stolen, accounts remain protected

User Access Levels Spec 1.4

Access Levels
A security measure that restricts what files and systems each user can access based on their role. Not everyone needs access to everything.
Permission Types
  • Read: Can view the file but not change it
  • Write: Can modify or add to the file
  • Execute: Can run the program/script
  • Delete: Can remove the file
💡 Example Scenario

In a school: Students can READ assignment briefs but cannot WRITE to them. Teachers can READ and WRITE. Admins have full access including DELETE. This way, if a student's account is compromised, the hacker can't modify important files.

Penetration Testing Spec 1.4

Penetration Testing
A simulated cyber attack on a system to find security weaknesses before real hackers do. Often performed by "ethical hackers."
Type Description Real-world analogy
White Box Tester has full knowledge of the system (source code, network diagrams) Like a security guard testing their own building
Black Box Tester has no prior knowledge – simulates real external attack Like a burglar trying to break in with no inside info
Grey Box Tester has partial knowledge Like a former employee testing security

1.5 Systems Software

Operating System: Memory Management Spec 1.5.1

Memory Management
The OS function that controls how RAM is allocated to programs, ensuring efficient use and preventing programs from interfering with each other.
Key Functions
  • RAM Allocation: OS decides where in RAM each program's data is stored
  • Protection: Prevents one program from accessing another program's memory space (security + stability)
  • Multitasking: Swaps data in and out of RAM to allow multiple programs to run "simultaneously"
  • Virtual Memory: Uses part of the hard drive as "overflow" RAM when physical RAM is full
💡 Why This Matters

Without memory management, opening a new browser tab could corrupt your Word document! The OS keeps each program's data separate and manages the limited RAM space.

Peripheral Management Spec 1.5.1

Device Drivers
Small programs that allow the operating system to communicate with hardware devices (printers, graphics cards, webcams, etc.).
Buffers
Temporary memory areas used to manage the speed difference between fast components (CPU) and slow devices (printer). Data is held in the buffer until the device is ready.
Example: Print Buffer
  • You click "Print" – the document is sent to the print buffer instantly
  • The CPU is now free to do other work
  • The printer slowly takes data from the buffer at its own pace
  • Without a buffer, you'd have to wait for the printer before doing anything else!

1.6 Ethical, Legal, Cultural & Environmental

Cultural Impacts Spec 1.6

Digital Divide
The gap between those who have access to technology and those who don't. Caused by factors like age, wealth, location (rural vs urban), and education.
Key Cultural Issues
  • Censorship: Governments controlling what citizens can access online (e.g., blocking social media)
  • Globalisation: Technology spreading Western culture globally – potentially eroding local cultures and languages
  • Monitoring/Surveillance: Cultural shift toward accepting tracking (CCTV, phone location) in exchange for convenience or security
  • Social isolation: Online interactions replacing face-to-face community connections

Computer Misuse Act 1990 Spec 1.6

Computer Misuse Act (1990)
UK law designed to combat hacking and cybercrime. Makes it illegal to access or modify computer systems without authorisation.
The 3 Offences
  • Section 1: Unauthorised access to computer material (basic hacking – accessing files you shouldn't)
  • Section 2: Unauthorised access with intent to commit further crime (hacking to steal money or data)
  • Section 3: Unauthorised modification of computer material (creating/spreading viruses, ransomware, deleting files)
💡 Common Exam Scenario

"A student guesses their friend's password and reads their emails" = Section 1 offence
"A hacker installs ransomware on a hospital network" = Section 3 offence

Privacy Spec 1.6

Privacy Concerns
  • Cookies: Small files that track browsing behaviour – used for targeted advertising
  • Algorithms: Social media uses your data to show personalised content (filter bubbles)
  • Consent: Ethical requirement to inform users HOW their data is being collected and used
  • Data Protection: Users have the right to see what data is held about them and request deletion
