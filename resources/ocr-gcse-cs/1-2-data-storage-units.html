Interaction 3 — Logic Hook

If 1 switch = 1 Bit, and a single page of text needs about 16,000 bits to store — why do we need units like "Kilobytes" and "Megabytes"? What's your best guess?

Because the numbers get unmanageably large, unbelievably fast. A 1-page document is already 16,000 bits. A typical photo is tens of millions of bits. A modern SSD holds trillions. Writing those numbers out in full would be impractical — just like a geographer doesn't measure the distance to the moon in millimetres. We group bits into larger, named units so that we can talk about data at a human scale. That's exactly what Kilobytes, Megabytes, and beyond are for.

📐 The Data Unit Hierarchy

Here is the complete scale — from a single binary switch to a petabyte. This table is your foundation for every conversion in this topic.

OCR Rule ÷ 1,000 going UP — × 1,000 going DOWN

To convert to a larger unit (e.g. Bytes → KB): divide by 1,000. To convert to a smaller unit (e.g. GB → MB): multiply by 1,000. OCR uses 1,000 for exam calculations — not 1,024.

Unit Abbreviation Equivalent Real-World Scale
Bit b 1 binary digit (0 or 1) 1 switch in an SSD
Nibble — 4 Bits One hexadecimal digit
Byte B 8 Bits One character of text
Kilobyte KB 1,000 Bytes A short email or text file
Megabyte MB 1,000 Kilobytes A typical photo (2–5 MB)
Gigabyte GB 1,000 Megabytes An HD film (~4 GB)
Terabyte TB 1,000 Gigabytes A large HDD (1–4 TB)
Petabyte PB 1,000 Terabytes Data centre scale storage
⚠️ 1,000 vs 1,024 — Know Both

For your OCR exam: always use 1,000 (the SI standard). However, you should know that operating systems (like Windows) often use 1,024 — this is why a "500 GB" hard drive shows as slightly less in File Explorer. If an exam question specifies 1,024, use that. Otherwise, default to 1,000.

🔗 The Direction Rule in Plain English

Going UP to a bigger unit? The number gets smaller → divide.
"5,000 MB ÷ 1,000 = 5 GB"

Going DOWN to a smaller unit? The number gets bigger → multiply.
"3 GB × 1,000 = 3,000 MB"

💡 The Aha Moment

Converting data units is exactly like converting grams to kilograms. You are just moving the decimal point. 1,000 grams becomes 1.0 kg — 1,000 bytes becomes 1.0 KB. The rule is identical at every level of the hierarchy. Once you see this, conversions stop being maths and start being pattern recognition.

📝 Notes for This Section

The Conversion Rule — Visualised

  • 1,000g = 1 kg → 1,000 Bytes = 1 KB
  • 1,000 kg = 1 tonne → 1,000 KB = 1 MB
  • The same ÷1,000 / ×1,000 rule applies at every level
  • Moving to a bigger unit = number gets smaller (÷1,000)
  • Moving to a smaller unit = number gets bigger (×1,000)

❌ Doesn't Fit — Here's Why

Total = 1.5 GB + 1.0 GB + 2.1 GB = 4.6 GB. The phone only has 4 GB free. You're 0.6 GB (600 MB) over the limit. You'd need to drop one of the items — probably the video files or the game.

🤔 Extension: What Would Fit?

If you removed the Videos (1.5 GB), the remaining total would be 1.0 + 2.1 = 3.1 GB — which would fit. Or you could keep all the videos and photos (2.5 GB) but skip the game. This kind of trade-off is exactly the decision people make every day on real devices.

📝 Notes for This Section

Smartphone Storage Calculation

  • 3 Videos: 500 MB × 3 = 1,500 MB = 1.5 GB
  • 500 Photos: 2 MB × 500 = 1,000 MB = 1.0 GB
  • 1 Game: 2,100 MB = 2.1 GB
  • Total: 1.5 + 1.0 + 2.1 = 4.6 GB
  • Free space = 4 GB → Does NOT fit (0.6 GB over)

📝 Full Topic Summary — Units of Data Storage

The Bit

  • The smallest unit of data — a single binary digit: 0 or 1
  • Physically represented by an On/Off switch in storage hardware

The Full Hierarchy (OCR J277)

  • Bit — 1 binary digit
  • Nibble — 4 Bits
  • Byte (B) — 8 Bits
  • Kilobyte (KB) — 1,000 Bytes
  • Megabyte (MB) — 1,000 KB
  • Gigabyte (GB) — 1,000 MB
  • Terabyte (TB) — 1,000 GB
  • Petabyte (PB) — 1,000 TB

The Conversion Rule

  • Moving to a larger unit → ÷ 1,000
  • Moving to a smaller unit → × 1,000
  • OCR uses 1,000 (SI standard) — not 1,024
  • 1,024 is technically correct (binary) and used by operating systems — acknowledge this but use 1,000 in calculations

Real-World File Sizes

  • Text file (1 page) ≈ 16 KB
  • Photo ≈ 2–5 MB
  • Song (MP3) ≈ 3–5 MB
  • HD video file ≈ 4–8 GB
  • Large game ≈ 1–100 GB

Exam Technique: Storage Calculation Questions

  • Convert all items to the same unit first (usually GB)
  • Add totals, then compare to available space
  • Show your working: 500 MB × 3 = 1,500 MB = 1.5 GB
  • State a clear conclusion: "does / does not fit because…"

The Analogy

  • Data units ↔ metric weight units
  • 1,000 g = 1 kg → 1,000 Bytes = 1 KB (same division)
  • Once you can do weight conversions, you can do data conversions
