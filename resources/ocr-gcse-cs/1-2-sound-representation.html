01 · ANALOGUE VS DIGITAL

Analogue vs Digital

Sound exists in the real world as a pressure wave. Computers store only binary numbers. These two things are fundamentally incompatible — so the wave must be converted.

CLICK EACH BUTTON TO REVEAL ▼

ANALOGUE

An analogue signal is continuous — it flows smoothly through every possible value with no gaps. Sound waves, voltage from a microphone, and light intensity are all analogue. On a graph, the amplitude can be any real number at any point in time.

  • Can take infinite values between any two points
  • Examples: microphone voltage, vinyl records, radio waves
  • Cannot be stored directly in a binary computer
DIGITAL

A digital signal is discrete — it can only hold specific values at specific times. In a binary system, those values are combinations of 0s and 1s. The signal looks like a staircase, not a smooth curve.

  • Only finite values — determined by bit depth
  • Examples: MP3 files, CDs, WAV files, digital radio
  • Can be stored, copied and transmitted without quality loss

❌ Digital audio is always an approximation of the original analogue wave. Because we take a finite number of measurements at finite precision, some information is inevitably lost. High quality settings minimise this loss — they don't eliminate it.
  • Analogue signal = continuous, smooth, infinite precision (e.g. microphone voltage)
  • Digital signal = discrete, stepped, stored as binary numbers (e.g. MP3 file)

  • Advantage of digital: perfect copies, no degradation, can be stored in computers
02 · ADC & SAMPLING

The ADC Process & Sample Rate

An Analogue-to-Digital Converter (ADC) transforms a continuous sound wave into a stream of binary numbers through three sequential stages.

CLICK EACH BUTTON TO REVEAL ▼

SAMPLING

The ADC measures the amplitude of the wave at regular time intervals. Each measurement is called a sample. The number of samples taken per second is the sample rate, measured in Hertz (Hz).

Sample RateSamples/secUse case
8,000 Hz8,000Phone calls (voice only)
44,100 Hz44,100CD / music streaming standard
96,000 Hz96,000Studio / professional recording
Effect on Quality

Higher sample rate → more measurements per second → the wave shape is captured with more accuracy → better quality audio. Lower sample rate → gaps between measurements → high-frequency detail is missed → audio sounds muffled or distorted.

QUANTISATION

The exact amplitude value at each sample is rounded to the nearest available whole-number level. The number of available levels is determined by the bit depth: 2n levels for n-bit depth.

This rounding introduces a small error called quantisation noise. With low bit depth (few levels), rounding errors are large — the digital wave looks like a rough staircase. With high bit depth, rounding errors are tiny and the staircase closely follows the original curve.

ENCODING + DAC

Encoding: Each quantised amplitude level is converted to its binary equivalent. For 8-bit audio, level 200 would be stored as 11001000. The stream of binary values forms the audio file.

Playback (DAC): A Digital-to-Analogue Converter reads the binary values and reconstructs an electrical signal that drives the speaker. The reconstructed wave is always an approximation — but at CD quality it is indistinguishable to the human ear.

NYQUIST — WHY 44,100 HZ?

The Nyquist theorem states that to accurately reproduce a sound, the sample rate must be at least twice the highest frequency in that sound.

Human hearing reaches up to approximately 20,000 Hz (20 kHz). Therefore the minimum sample rate to capture all audible frequencies is 2 × 20,000 = 40,000 Hz. CD quality uses 44,100 Hz — slightly above Nyquist minimum, with some headroom.

⚠️ Below Nyquist Rate

If the sample rate is less than twice the highest frequency in the sound, the reconstructed wave will contain aliasing — false lower-frequency artefacts that were not in the original signal.

✓ CHECK — Q2
During quantisation, the exact amplitude value 6.73 is rounded to the nearest level, which is 7. What is the small error introduced by this rounding called?
AAliasing — a frequency artefact caused by sampling below the Nyquist rate
BSampling error — the difference between the measured value and the true amplitude
CQuantisation noise — the small inaccuracy introduced when an amplitude is rounded to the nearest available discrete level
DEncoding error — a mistake made when converting the amplitude value into its binary equivalent
✅ Correct! Quantisation noise = the rounding error at each sample. More bit depth → more levels → levels closer together → smaller rounding error → less noise. Aliasing is a different issue (too-low sample rate).
❌ The rounding error introduced during quantisation is called quantisation noise. Aliasing is caused by too low a sample rate. Encoding is converting the level to binary — a separate stage from rounding.
  • ADC = Analogue-to-Digital Converter — 3 stages: Sampling → Quantisation → Encoding
  • Sample rate = samples per second (Hz). CD quality = 44,100 Hz
  • Quantisation = rounding amplitude to nearest level. Error = quantisation noise
  • Nyquist: sample rate must be ≥ 2× the highest frequency in the sound
  • DAC = Digital-to-Analogue Converter — reverse process for playback
03 · BIT DEPTH

Bit Depth & Sample Resolution

Bit depth (also called sample resolution) controls how many amplitude levels are available for each measurement. More bits = finer precision = less quantisation noise.

CLICK EACH BUTTON TO REVEAL ▼

BIT DEPTH
Bit DepthAmplitude Levels (2ⁿ)Use Case
8-bit256Old games, speech
16-bit65,536CD quality — music
24-bit16,777,216Professional studio
Effect of Bit Depth

Higher bit depth → more levels → levels packed closer together → smaller quantisation rounding error → cleaner, more accurate sound. Lower bit depth → few levels → large gaps between them → aggressive rounding → audible hiss and distortion.

SAMPLE RATE ≈ PIXELS / COLOUR DEPTH

Think of a digital photograph. The number of pixels determines how much spatial detail is captured — more pixels = finer resolution. This is like sample rate (more measurements per second = finer time resolution).

But each pixel also has a colour depth — how many colours it can represent. More bits per pixel = more available colours = smoother gradients and less banding. This is like bit depth — more bits per sample = more amplitude levels = less quantisation noise.

✓ CHECK — Q3
A sound engineer switches from 8-bit to 16-bit recording. By exactly how much does the number of available amplitude levels increase?
ADoubles — from 8 to 16 levels
BDoubles — from 128 to 256 levels
CIncreases 256× — from 256 levels (2⁸) to 65,536 levels (2¹⁶)
DIncreases by 8 — from 256 to 264 levels
✅ Correct! 8-bit = 2⁸ = 256. 16-bit = 2¹⁶ = 65,536. That is a 256× increase. Going from 8-bit to 16-bit doesn't just double the quality — it multiplies the precision by a factor of 256.
❌ 8-bit = 2⁸ = 256 levels. 16-bit = 2¹⁶ = 65,536 levels. 65,536 ÷ 256 = 256× more levels — not just double. This is the power of exponential growth from adding bits.
  • Bit depth = number of bits per sample. Levels available = 2^(bit depth)
  • 8-bit = 256 levels · 16-bit = 65,536 levels · 24-bit = 16.7 million levels
  • Higher bit depth → finer amplitude precision → less quantisation noise
  • Analogy: bit depth ≈ colour depth in images (more colours = smoother gradients)

04 · FILE SIZE

File Size Calculation

Better quality always means more data. You must be able to calculate the uncompressed file size of any audio recording and understand the quality–storage trade-off.

CLICK EACH BUTTON TO REVEAL ▼

FILE SIZE FORMULA
Uncompressed File Size
Bits = Duration(s) × Sample Rate(Hz) × Bit Depth × Channels ÷ 8 → bytes ÷ 1,000 → KB ÷ 1,000 → MB
  • Duration = length in seconds
  • Sample Rate = samples per second (Hz)
  • Bit Depth = bits per sample
  • Channels = 1 (mono) or 2 (stereo)
  • The formula gives bits — divide by 8 to get bytes

  • Formula: Duration(s) × Sample Rate(Hz) × Bit Depth × Channels = BITS
  • ÷ 8 → bytes · ÷ 1,000 → KB · ÷ 1,000 → MB
  • Stereo = 2 channels · Mono = 1 channel
  • Higher quality (more samples, more bits) = larger file size — always
