Question 1

Where does the CPU look to find instructions and data when it is running a program?

The CPU fetches instructions and data from RAM (Random Access Memory). During the Fetch phase of the FDE cycle, the MAR sends an address to RAM and the instruction returns via the MDR.
Question 2

Name two factors that affect the overall performance of a CPU.

Any two from: Clock Speed (how many cycles per second), Number of Cores (how many processors on the chip), and Cache Size (how much fast-access memory is built in).
Question 3

RAM is described as volatile. What does volatile mean in this context?

Volatile means the data stored in RAM is lost when the power is switched off. RAM requires a constant power supply to maintain its contents.

💾 What is Cache Memory?

Every time the CPU needs an instruction or a piece of data, it has to fetch it from RAM. RAM can hold gigabytes of data — but that size comes at a cost: speed. RAM is significantly slower than the CPU. If the CPU had to wait for RAM every single cycle, billions of clock cycles would be wasted sitting idle.

Cache memory solves this problem.

Key Term Cache Memory

A small, extremely fast type of memory built directly into (or very close to) the CPU. It stores copies of the most frequently used instructions and data. Before the CPU looks in RAM, it checks cache first.

The logic is simple: if you're likely to need the same data again very soon, keep a copy somewhere fast. Cache is that "somewhere fast."

🤔 Why is Cache so Much Faster than RAM?

Cache uses a type of memory called SRAM (Static RAM). Each bit is stored using a small circuit of transistors that maintains its state without needing to be constantly refreshed. RAM uses DRAM (Dynamic RAM), which stores bits as tiny charges in capacitors that leak and need refreshing thousands of times per second — this refresh cycle introduces delay. SRAM is faster, but far more expensive to manufacture.

⚠️ Common Misconception

Cache does not store all the data from RAM. It only stores copies of recently or frequently used data. The CPU predicts what it will need next and pre-loads it. Cache is never a complete picture of what's in RAM — it's a smart, temporary shortcut.

Understanding Each Level

L1 Cache is the smallest but the fastest. It sits right inside each CPU core — physically on the same silicon die. Access time is typically 1–4 clock cycles. Think of it as the CPU core's own private, instant-access shelf.

L2 Cache is slightly larger and slightly slower. It may be per-core or shared between a small cluster of cores. It acts as a second chance before going all the way to L3 or RAM.

L3 Cache is the largest cache level, shared across all CPU cores. It's still much faster than RAM, but the sharing means it's slower than L1 and L2. On modern processors this can be tens of megabytes.

🔗 The Pattern to Remember

As the level number goes up: size increases, speed decreases, distance from the core increases. L1 is small, fast, close. L3 is large, slower, shared. RAM is huge, much slower, off the chip entirely.

💡 Where Does L2 and L3 Fit?

Extend the analogy: your L2 cache is a slightly larger notepad on the corner of your desk — more space than sticky notes, slightly further to reach, but still on the desk. How the Homework Gets Done

When you need a formula, you check your sticky notes first. If it's there — you keep working at full speed. In computing, this is called a Cache Hit.

If it's not on the sticky note, you have to pause, reach into your bag, and find the page. You lose several seconds. In computing, this is called a Cache Miss.

If you had a slightly larger sticky note, you could write more formulas on it. More formulas on the desk means fewer trips to the bag. Fewer trips to the bag means you finish your homework faster — even though your brain hasn't become any smarter or faster. This is exactly how a larger cache improves CPU performance.

⚡ Cache Hit vs Cache Miss

These are two of the most important terms for the exam. Let's be precise about what they mean — and then see them in action.

Key Term Cache Hit

When the CPU needs data and finds it already stored in cache. The data is returned directly from cache — no RAM access required. This is fast.

Key Term Cache Miss

When the CPU needs data and it is not in cache. The CPU must fetch it from the slower RAM. This causes the CPU to enter an idle (waiting) state, reducing performance. The fetched data is then copied into cache for future use.

🤔 Put This in Perspective

If L1 cache access were one second, RAM access would be around three to four minutes. Secondary storage (SSD) access would be roughly four days. This is why the CPU cannot afford to go to RAM every time it needs data — and why having more data in cache is so valuable.

🏎️ Cache Size and CPU Performance

Now we return to exam language. You need to be able to explain — precisely — why a larger cache improves performance. Saying "it's faster" is not enough. Here is the causal chain:

①

A larger cache can store more frequently used instructions and data.

↓
②

The probability of the CPU finding data in cache (Cache Hit Rate) increases.

↓
③

The CPU spends less time in an idle state waiting for data to arrive from the slower RAM.

↓
④

The CPU completes tasks faster — even if its clock speed is identical.

Key Term Cache Hit Rate

The percentage of memory accesses that are successfully served by the cache (rather than requiring a trip to RAM). A higher cache hit rate means fewer cache misses, less idle time, and better performance.

⚠️ Exam Language Alert

Don't just write "larger cache means faster CPU." The marks come from the chain: larger capacity → stores more frequently-used instructions → higher cache hit rate → CPU idles less waiting for RAM → task completes faster. Examiners want causation, not just correlation.

  • Larger cache → stores more instructions/data
  • More stored → higher Cache Hit Rate
  • Higher hit rate → fewer trips to RAM
  • Fewer RAM trips → CPU idles less
  • Less idling → faster task completion

✅ Model Answer (4 marks)

Computer B has a larger cache (8 MB compared to 4 MB), which means it is able to store a greater volume of frequently used instructions and data. This increases the cache hit rate — the proportion of memory accesses where the required data is found in cache rather than RAM.

As a result, the CPU spends less time in an idle state waiting for data to be fetched from the slower RAM. Since both CPUs run at the same clock speed, the difference in performance is entirely due to cache efficiency. Computer B's data fetch pipeline is more efficient, so it completes the task in less time.

🎯 The 4 Marks Mapped

Mark 1: Computer B has a larger cache, so it can store more frequently used data/instructions.

Mark 2: This increases the cache hit rate.

Mark 3: The CPU spends less time idle / waiting for data from RAM.

Mark 4: Since clock speed is identical, the difference in completion time is due to cache efficiency alone.

🧩 Critical Thinking

🤔 The Big Question

"If cache is so much faster than RAM — why don't we just build all of our RAM out of cache memory?"

💡 Two Reasons — Both Examinable

1. Cost: Cache uses SRAM, which requires more transistors per bit than the DRAM used in RAM. Manufacturing SRAM is significantly more expensive per gigabyte. A typical computer has 8–32 GB of RAM; making that volume out of SRAM would cost thousands of pounds per machine.

2. Physical size: SRAM takes up considerably more space on the silicon die than DRAM. There is a hard physical limit to how much cache can fit on a CPU chip. You cannot simply scale it to gigabytes without making the chip impractically large and power-hungry.

⚠️ The Fundamental Trade-off

This is the core tension in CPU design: faster memory is always smaller and more expensive. The entire cache hierarchy exists as a solution to this trade-off — getting the benefit of speed where it matters most (close to the core) while still having affordable, large-capacity memory (RAM) for the full dataset.
