📦 What Are Variables?

Definition Variable

A named location in memory where data can be stored and changed during program execution.

Variables are the workhorses of programming. They let your program remember information and update it as things change.

# Creating variables and changing their values score = 0 # Start with 0 score = 100 # Player earns points score = 150 # Player earns more points playerName = "Alex" isAlive = True

✏️ Naming Variables & Assignment

Choosing good variable names and understanding assignment is essential for writing clear, working code.

Definition Identifier

The name given to a variable, constant, or other element in a program. Must follow specific rules.

Naming Rules

✓ Start with a letter: score
✗ Start with number: 1stPlace
✓ Use underscores: player_name
✗ Use spaces: player name
✓ Use camelCase: playerName
✗ Use symbols: player@name
✓ Be descriptive: totalScore
✗ Use keywords: print

Assignment Operator

variableName = value

The = sign means "assign" or "store this value in". It does NOT mean "equals" in programming!

# Assignment examples age = 25 # Assign 25 to age name = "Sam" # Assign "Sam" to name price = 9.99 # Assign 9.99 to price age = age + 1 # Add 1 to age (now 26)
⚠️ Common Mistake

In programming, x = x + 1 is valid! It means "take the current value of x, add 1, and store the result back in x". This looks strange mathematically but makes perfect sense as an instruction.

📝 Notes for This Section

Naming Rules

  • Must start with a letter (not a number)
  • No spaces — use camelCase or underscores
  • No special symbols (except underscore)
  • Can't use reserved keywords (print, if, while)
  • Make names descriptive

Assignment

  • = means "assign/store", not "equals"
  • x = x + 1 is valid — it updates x

🔒 What Are Constants?

Definition Constant

A named location in memory where data is stored and cannot be changed after it is set.

Constants store values that should never change during program execution. They protect important values from accidental modification.

# Constants are typically written in UPPERCASE MAX_LIVES = 3 TAX_RATE = 0.20 PI = 3.14159 SCHOOL_NAME = "Oakwood Academy" GRAVITY = 9.8

Why Use Constants?

🛡️
Protection
Prevents accidental changes to important values
📖
Readability
TAX_RATE is clearer than 0.20
🔧
Maintenance
Change once, updates everywhere
🎯
Clarity
UPPERCASE shows it shouldn't change
📦 Storage Box Analogy

A constant is like a sealed, labelled box. Once you put the value inside and seal it, you can read what's in it, but you can't change it. If tax rate is 20%, you don't want code accidentally changing it to 200%!

🤔 When to Use Constants?

Use constants for values that are:
• Fixed by rules: MAX_PLAYERS = 4, PASSING_GRADE = 40
• Mathematical: PI = 3.14159, GRAVITY = 9.8
• Configuration: TAX_RATE = 0.20, SCHOOL_NAME = "..."

Constants

  • Definition: Named value that cannot change
  • Convention: Use UPPERCASE_WITH_UNDERSCORES
  • Purpose: Protect fixed values, improve readability
  • Examples: MAX_LIVES, TAX_RATE, PI

🏷️ Introduction to Data Types

Definition Data Type

A classification that specifies what kind of value a variable can hold and what operations can be performed on it.

Just like containers come in different shapes for different items, data types tell the computer what kind of data to expect and how to handle it.

📦 Storage Box Analogy

Data types are like different shaped containers. You wouldn't put liquid in a cardboard box or books in a jar! Similarly, you can't do maths with text or spell-check a number. The right type for the right data.

⚠️ Why Types Matter

"5" + "3" gives "53" (joining text), but 5 + 3 gives 8 (adding numbers). The data type determines what operations do!

📝 Strings, Characters & Integers

Data Type String

A sequence of characters (text) enclosed in quotation marks. Can include letters, numbers, symbols, and spaces.

# String examples name = "Alice" message = "Hello, World!" postcode = "SW1A 1AA" # Numbers as text! phone = "07123456789" # Stored as text empty = "" # Empty string
🤔 When to Use Strings

Use strings for: names, addresses, messages, postcodes, phone numbers (you don't do maths with them!), and any text the user enters or sees.

Data Type Character

A single character enclosed in quotation marks. Just one letter, digit, or symbol.

# Character examples grade = "A" initial = "J" symbol = "@" digit = "7" # Single digit as character space = " " # A space is a character
🤔 String vs Character

A string is a sequence of characters (any length), while a character is exactly one symbol. Think of a character as a string with length 1. In Python, there's no separate character type — single characters are just strings of length 1.

Data Type Integer

A whole number (positive, negative, or zero) without any decimal point.

# Integer examples age = 25 score = 1500 lives = 3 temperature = -5 # Negative integer count = 0 # Zero is an integer year = 2024
🤔 When to Use Integers

Use integers for: counting things, ages, scores, years, quantities, and any whole number you might do calculations with.

⚠️ String vs Integer Trap

"25" is a STRING (text), but 25 is an INTEGER (number).
"25" + "10" = "2510" (joins text)
25 + 10 = 35 (adds numbers)

📝 Notes for This Section

Strings

  • Text in quotation marks: "Hello"
  • Use for: names, messages, postcodes, phone numbers
  • Can be joined (concatenated): "Hi" + "!" = "Hi!"

Integers

  • Whole numbers: 42, -7, 0
  • Use for: counting, ages, scores, years, quantities
  • Can do maths: 5 + 3 = 8

🔣 Floats & Booleans

Data Type Float (Real)

A number that contains a decimal point. Also called a "real number" in some contexts.

# Float examples price = 19.99 temperature = 21.5 pi = 3.14159 percentage = 85.5 weight = 72.3 balance = -15.50 # Negative float
🤔 When to Use Floats

Use floats for: prices (£19.99), measurements (72.3 kg), temperatures (21.5°C), percentages (85.5%), and any value that needs decimal precision.

Data Type Boolean

A data type with only TWO possible values: True or False.

# Boolean examples isLoggedIn = True gameOver = False hasDiscount = True isAdult = age >= 18 # True if age is 18+ isEmpty = stock == 0 # True if stock is 0
🤔 When to Use Booleans

Use booleans for: yes/no questions, on/off states, conditions (isLoggedIn, hasPermission, gameOver), and anything that's either true or false.

📦 Storage Box Analogy

A boolean is like a light switch box — it can only be ON (True) or OFF (False). There's no in-between, no "maybe", just two states.
