SECTION 02

What Is Virtual Memory?

📝 Definition

Virtual memory is a reserved area of secondary storage (hard disk or SSD) that the operating system uses temporarily as if it were RAM. It is not real physical main memory — it is a section of the disk that extends the apparent size of RAM.

The key word is "virtual" — it behaves like extra RAM from the running program's perspective, but it is actually space on the hard drive or SSD. The operating system manages the illusion entirely — programs do not need to know their data is temporarily on disk.

💾 RAM (Real Memory)

  • Physical — actual memory chips on the motherboard
  • Fast — nanosecond access time
  • Volatile — loses data when power is off
  • Limited — fixed amount installed in the computer

🗄️ Virtual Memory (on disk)

  • On secondary storage — part of the hard drive / SSD
  • Slow — millisecond access (1000× slower than RAM)
  • Non-volatile — survives power loss
  • Flexible — OS allocates as much disk space as needed
🤔 Analogy — Your Desk and a Filing Cabinet

Think of RAM as your desk — you can only work on what's on the desk. The hard drive is the filing cabinet in the corner of the room. When your desk is full, you move some papers to the filing cabinet to make room. You can still work — but fetching papers from the cabinet takes much longer than picking them up from the desk.

SECTION 03

Why Is Virtual Memory Needed?

Modern computers run a multi-tasking operating system — many programs are open at the same time. Each open program occupies space in RAM.

📝 The Core Problem

When the total memory needed by all open programs exceeds the physical RAM available, the computer faces a choice: crash with a "not enough memory" error, or find a way to cope. Virtual memory is that solution.

Without virtual memory, opening too many programs at once would simply fail. With virtual memory, the operating system can keep running — it manages which data stays in fast RAM and which is temporarily parked on disk.

Example — A Typical Student's Laptop

Virtual memory solves this by allowing the OS to park the inactive programs (like minimised PowerPoint) on disk, freeing RAM for what the user is actively doing.

🔗 Link to the OS

Managing virtual memory is one of the key jobs of the operating system. The OS decides which data in RAM is least recently used, moves it to virtual memory on the disk, and tracks where everything is so it can retrieve it when needed. This is all transparent to the user.

How Virtual Memory Works — The Swapping Process

When RAM is full and a new program (or more data) needs to be loaded, the OS performs a swap. This is a two-step move: something leaves RAM, and something enters RAM.

Step 0 / 4

The process the OS follows:

  • Step 1 — RAM fills up. All 4 RAM slots are occupied by running programs.
  • Step 2 — New program needs loading. Microsoft Word needs to open but there is no free RAM.
  • Step 3 — OS identifies the least active program. PowerPoint is minimised and hasn't been touched for 10 minutes — it is chosen to be swapped out.
  • Step 4 — PowerPoint is moved to virtual memory on disk. This frees a RAM slot.
  • Step 5 — Word is loaded into the freed RAM slot and can now be used by the CPU.
  • Step 6 — If PowerPoint is later reopened, its data is swapped back from disk into RAM, and something else (e.g. Word if it's now idle) is swapped out.
⚠️ The Speed Penalty

Every time a swap happens, the system must read from or write to the hard disk or SSD — which is dramatically slower than RAM. A RAM access takes nanoseconds. A disk access takes milliseconds — roughly 1,000 to 100,000 times slower. This is why virtual memory works but comes at a significant performance cost.

✓ CHECK — Q3
When the OS moves data from RAM to virtual memory to free up space, what is this process called, and where does the freed space go?
ADefragmentation — the freed space is merged with other free areas to create a larger contiguous block
BSwapping (or paging) — the data is written to the virtual memory area on secondary storage, and the space it occupied in RAM becomes available for new data
CCaching — the data is copied into a faster memory layer between the CPU and RAM for quicker future access
DPartitioning — RAM is divided into sections so different programs cannot interfere with each other's memory
✓ Correct. The process is called swapping (or paging at the OS level, though J277 does not require that term). The OS moves program data from RAM to the virtual memory area on disk. The RAM space that was occupied becomes free and available to load new data or a new program that the CPU needs.
✗ The process is called swapping. Data moves from RAM to the virtual memory area on the hard disk/SSD — freeing up space in RAM. Defragmentation is a disk maintenance process (unrelated). Caching moves data to a faster layer, the opposite direction. Partitioning divides storage space, not memory management.
✏️ Book Notes
  • RAM full → OS swaps least-active program to virtual memory (disk)
  • Freed RAM slot → used to load the program the CPU needs now
  • Program swapped back to RAM when user opens it again
  • Each swap = disk access = much slower than staying in RAM

SECTION 06

Impact on Performance

Virtual memory allows more programs to run, but it comes at a significant performance cost. The reason is simple: disk access is orders of magnitude slower than RAM access.

RAM
~10 ns — near-instant
SSD (virtual memory)
~100,000 ns
HDD (virtual memory)
~10,000,000 ns

Note: RAM is so much faster that HDD virtual memory barely shows on this scale. Bar widths are simplified for illustration.

When the OS frequently needs to swap data in and out of virtual memory, the user notices the system slowing down, freezing briefly, or becoming unresponsive. The hard drive activity light may be constantly on. Tasks that would take milliseconds in RAM can take seconds when virtual memory is involved.

⚠️ Thrashing

Thrashing occurs when the system is spending more time swapping data between RAM and virtual memory than it is actually running programs. This happens when RAM is severely insufficient for the workload — the OS is constantly swapping, and as soon as one program is loaded, another needs to be swapped in, creating a vicious cycle. The computer becomes almost unusable.

🤔 Recognising Thrashing

Signs a computer is thrashing: everything is very slow, the hard drive is constantly active (light on, whirring), moving between windows takes a long time, programs seem to "freeze" momentarily when you switch to them. The system works, but barely. The solution is almost always to add more RAM.

✓ CHECK — Q6

BWhen the hard disk runs out of space for virtual memory and programs begin to crash
CWhen the system spends more time swapping data between RAM and virtual memory than actually executing programs — caused by severely insufficient RAM for the current workload
DWhen RAM is so fast that it causes the CPU to request data faster than secondary storage can supply it, creating a bottleneck
✓ Correct. Thrashing is the condition where the system is almost entirely occupied with the overhead of swapping data in and out of virtual memory, leaving little time to actually run programs. It is caused by RAM being seriously insufficient for the number and size of programs running. The performance impact is severe — the system becomes near-unusable.
✗ Thrashing is specifically about the performance collapse caused by constant swapping between RAM and virtual memory. It is not about CPU temperature (Option A), disk space running out (Option B — though that's a related concern), or RAM being too fast (Option D — that doesn't make sense). SECTION 07

Solving the Problem — Increasing RAM

The long-term solution to performance issues caused by virtual memory is straightforward: add more physical RAM. More RAM means fewer swaps are needed — the OS can keep more programs in fast memory simultaneously.

With insufficient RAM

  • OS constantly swapping to virtual memory
  • Disk I/O (reading/writing to VM) creates bottleneck
  • Switching between programs feels slow or laggy
  • Risk of thrashing if demand far exceeds RAM

With more RAM installed

  • More programs fit in RAM simultaneously
  • Fewer (or no) swaps needed → less disk I/O
  • Switching between programs is instant
  • Virtual memory rarely or never invoked
📝 The Relationship to State Clearly in Exams
  • Increasing RAM → less reliance on virtual memory
  • Less reliance on virtual memory → fewer disk accesses
  • Fewer disk accesses → faster overall system performance
  • Virtual memory allows more programs to run but always at a performance cost compared to keeping everything in RAM
🤔 Virtual Memory on an SSD vs HDD

Virtual memory stored on an SSD is significantly faster than virtual memory on a traditional HDD — SSDs have no moving parts and much shorter access times. This is why modern laptops with SSDs feel less "sluggish" when virtual memory kicks in compared to older HDD machines. However, SSD virtual memory is still dramatically slower than real RAM — it is a mitigation, not a solution.
