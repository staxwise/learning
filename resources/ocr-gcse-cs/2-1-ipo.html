Every computer program follows the same fundamental pattern: it takes in data, does something with it, and produces a result. This is called the Input-Process-Output (IPO) model.

📥 What Are Inputs?

📥 Definition

Data or information supplied to the program or algorithm.

Examples: Usernames, passwords, scores, sensor readings, file uploads, menu selections, slider values, GPS coordinates.

Inputs are the raw materials your program needs. Without inputs, a program has nothing to process!

Where Do Inputs Come From?

👤

Users

Typing, clicking, touching, speaking

📁

Files

Reading from documents or databases

📡

Sensors

Temperature, motion, light, pressure

🌐

Other Systems

APIs, network data, other programs

🤔 Design Considerations

Variable names: Use descriptive names like customerAge not x
Data types: String, Integer, Float, or Boolean?
Validation: What if the user enters invalid data?

⚙️ What Are Processes?

⚙️ Definition

The actions, calculations, or transformations performed on the input data.

Examples: Checking passwords, calculating totals, sorting data, updating records, checking availability, converting formats.

Processes are the brain of your program — where inputs are transformed into meaningful outputs.

Types of Processing

🧮

Calculations

Maths operations, formulas, totals

⚖️

Comparisons

Checking conditions, validation

🔄

Data Manipulation

Sorting, filtering, formatting

🔀

Logic Decisions

If/else branches, loops

⚠️ Order Matters!

The sequence of processes is important. You can't apply a discount before calculating the subtotal, or check a password before the user types it!

📝 Notes for This Section

Processes

  • Definition: Actions/transformations performed on data
  • Types: Calculations, comparisons, manipulation, logic
  • Remember: Sequence matters — some steps depend on others

📤 What Are Outputs?

📤 Definition

The final result or information produced by the program.

Examples: Confirmation messages, error messages, calculated values, displays, sounds, printed documents, emails.

Outputs are the results — what the user sees, hears, or receives. This is the whole point of the program!

Types of Output

🖥️

Visual

Text, images, graphs, animations

🔊

Audio

Sounds, alerts, spoken feedback

💾

Data

Saved files, database updates

📧

Communication

Emails, notifications, API responses

🤔 Good Output Design

Be specific: "Password must be 8+ characters" not just "Error"
Format properly: Show prices as £12.99, not 12.987654
Be helpful: Tell users what to do next

📝 Quick Reference

How to Identify Components

  • Input: Data going INTO the system (user types, sensor reads)
  • Process: Something DONE to data (calculate, check, sort)
  • Output: Results coming OUT (display, print, send)

🎬 Example: Cinema Ticket System

Let's analyse a complete system to see IPO in action.

🎬 Cinema Self-Service Kiosk

A cinema has self-service kiosks where customers browse films, select seats, and buy tickets. The system checks availability, calculates prices with discounts, and produces tickets.

📥 Inputs
  • Film selection
  • Showtime choice
  • Number of tickets
  • Ticket types (adult/child)
  • Seat preferences
  • Payment details
⚙️ Processes
  • Check seat availability
  • Calculate ticket prices
  • Apply age discounts
  • Calculate total cost
  • Process payment
  • Reserve the seats
📤 Outputs
  • Available seats display
  • Price breakdown
  • Payment confirmation
  • Printed tickets
  • Email receipt
  • Updated seat map
🤔 Variable Considerations

• filmTitle (String) — name of the selected film
• numAdultTickets (Integer) — count of adult tickets
• seatNumber (String) — e.g., "F12"
• totalCost (Float) — final price with pence

📝 Exam Tip

Identifying IPO Components

  • Ask: "Is this data going into the system?" → Input
  • Ask: "Is something being done to data?" → Process
  • Ask: "Is this a result shown/sent?" → Output
