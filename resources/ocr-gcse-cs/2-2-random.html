Adding unpredictability to your programs

Many programs need an element of unpredictability. Without it, games would be boring (imagine dice that always roll 6!), simulations would be useless, and passwords would be easy to guess.

Python provides a random module that makes generating random values easy.

🎲 Lucky Dip Analogy

Think of random number generation like a lucky dip at a fair. You reach into a bag without looking and pull out a prize. You don't know what you'll get — that's the point!

Python's random functions work the same way: you define the "bag" (the range of possible values), and Python picks one for you.

🎯 Why Use Random Numbers?

Random numbers are essential for adding unpredictability to programs. Here are the main uses:

🎮

Games

Dice rolls, card shuffling, enemy spawns, loot drops

🔬

Simulations

Modelling weather, traffic, disease spread, stock prices

🔐

Security

Password generation, encryption keys, verification codes

🌐

Networking

Random delays to prevent collisions, load balancing

Key Concept Unpredictability

The main purpose of random number generation is to add unpredictability — making it impossible to know what value will be generated next.

🤔 Think About It

Why would network systems use random delays? If multiple devices try to send data at exactly the same time, they collide. Random delays mean they retry at different times, reducing collisions.

📦 Importing the Random Module

Before you can use random functions, you need to import the random module. This tells Python to load the random number tools.

Syntax

import random

This line goes at the top of your program, before you use any random functions.

# Always import at the top of your program
import random

# Now you can use random functions
number = random.randint(1, 6)
print(number)
🎲 Lucky Dip Analogy

Think of import random as getting the lucky dip bag ready. You need the bag before you can dip your hand in!

⚠️ Common Mistake

Forgetting to import! If you try to use random.randint() without importing first, you'll get a NameError.

🔢 Random Integers: randint()

Use randint() to generate a random whole number (integer) within a range.

Syntax

random.randint(min, max)

Both min and max are inclusive — they can both be chosen.

Example: Simulating a Dice Roll

import random

# Roll a 6-sided dice
dice = random.randint(1, 6)
print("You rolled:", dice)
Output
Important Inclusive Range

randint(1, 6) can return 1, 2, 3, 4, 5, or 6. Both endpoints are included!

🎲 Lucky Dip Analogy

With randint(1, 6), imagine a bag containing tickets numbered 1 to 6. You reach in and pull one out at random — any of the six tickets is equally likely.

📝 Notes for This Section

randint()

  • Generates random integers (whole numbers)
  • Syntax: random.randint(min, max)
  • Range is inclusive — both min and max can be returned
  • Example: randint(1, 6) → dice roll (1-6)

📊 Random Floats: uniform()

Use uniform() to generate a random decimal number (float) within a range.

Syntax

random.uniform(min, max)

Both min and max are inclusive. Returns a float (decimal).

Example: Random Temperature

import random

# Generate random temperature between 15.0 and 25.0
temp = random.uniform(15.0, 25.0)
print("Temperature:", temp)
Output
🤔 When to Use uniform() vs randint()

randint() — When you need whole numbers (dice, player count, questions)

uniform() — When you need decimals (temperature, prices, coordinates, percentages)

🎲 Lucky Dip Analogy

If randint() is like picking from numbered tickets, uniform() is like spinning a wheel that can land anywhere on a continuous scale — any decimal value is possible.

🎯 Random Choice: choice()

Use choice() to select a random item from a list.

Syntax

random.choice([item1, item2, item3])

Pass a list in square brackets. Returns one randomly selected item.

Example: Random Colour

import random

# Pick a random colour
colours = ["red", "blue", "green", "yellow"]
colour = random.choice(colours)
print("Selected:", colour)
Output

Example: Rock, Paper, Scissors

import random

# Computer makes a choice
computer = random.choice(["rock", "paper", "scissors"])
print("Computer chose:", computer)
Output
🎲 Lucky Dip Analogy

choice() is exactly like a lucky dip — you put specific items in a bag (the list), and Python picks one at random for you.

⚠️ Don't Forget the Square Brackets!

The items must be in a list (square brackets). random.choice("rock", "paper") won't work — it needs to be random.choice(["rock", "paper"])

choice()

  • Selects a random item from a list
  • Syntax: random.choice([items])
  • Items go in square brackets (a list)
  • Use for: colours, names, options, cards, directions

💡 Practical Examples

Example 1: Password Generator

import random

# Generate a 4-digit PIN
pin = random.randint(1000, 9999)
print("Your PIN:", pin)
Output

Example 2: Quiz Question Selector

import random

questions = ["What is 2+2?", "What is the capital of France?", "What year did WW2 end?"]
question = random.choice(questions)
print(question)
Output

Example 3: Simulation - Coin Flip Counter

import random

heads = 0
for i in range(100):
    flip = random.choice(["heads", "tails"])
    if flip == "heads":
        heads = heads + 1
print("Heads:", heads, "out of 100")
Output
🤔 Choosing the Right Function

randint() → Whole numbers (dice, PINs, scores)
uniform() → Decimals (prices, temperatures, percentages)
choice() → Pick from specific options (colours, names, directions)
