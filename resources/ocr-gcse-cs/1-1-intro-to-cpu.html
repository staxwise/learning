🧠 What is the CPU?

Key Term CPU (Central Processing Unit)

The component that processes instructions and controls all other parts of the computer. It fetches instructions, works out what they mean, and carries them out.

The CPU has two main jobs:

  1. Processing instructions — carrying out the operations that programs need to run
  2. Controlling the computer — coordinating all other components (memory, storage, input/output devices)
🍳 Kitchen Analogy

Think of the CPU as the head chef in a restaurant kitchen. The head chef:

• Reads orders (instructions) and works out what needs to be done

• Actually prepares the food (processes data)

• Coordinates the whole kitchen — telling other staff what to do and when

Without the head chef, nothing gets made. Without the CPU, nothing happens.

⚠️ Common Misconception

The CPU doesn't store your files or remember things long-term — that's the job of storage (like an SSD or hard drive) and RAM. The CPU just processes things in the moment.

📝 Where Do Instructions Come From?

The CPU processes instructions — but where do these instructions come from?

Every instruction the CPU follows was originally written by a human programmer. When programmers write code (in languages like Python, JavaScript, or C++), that code eventually gets converted into simple instructions the CPU can understand.

Key Term Program

A set of instructions that tells the computer what to do. Programs are written by humans using programming languages, then translated into a form the CPU can execute.

🍳 Kitchen Analogy

Instructions are like orders from customers. Someone (the programmer) writes down what they want. The kitchen (computer) receives the order, and the head chef (CPU) follows the instructions to make it happen.

Just as a chef follows recipes written by humans, the CPU follows programs written by humans.

The Journey of an Instruction

  1. A programmer writes code (e.g., print("Hello"))
  2. The code is translated into simple CPU instructions
  3. Instructions are stored in memory (RAM)
  4. The CPU fetches and executes each instruction, one by one
🤔 Think About It

Every app on your phone, every website you visit, every game you play — they all started as code written by humans. The CPU is just following those instructions, incredibly fast.

🎛️ The Control Unit (CU)

Key Term Control Unit (CU)

The component that manages and coordinates all CPU operations. It fetches instructions from memory, decodes them to understand what to do, and directs other components to carry out the work.

The Control Unit is like the manager of the CPU. It doesn't do the actual calculations — instead, it:

  • Fetches instructions from memory
  • Decodes instructions (works out what they mean)
  • Tells other components what to do and when
  • Controls the flow of data between CPU components
🍳 Kitchen Analogy

The Control Unit is like the kitchen manager who:

• Reads the order tickets (fetches instructions)

• Understands what each order needs (decodes)

• Tells the chef which station to use (directs the ALU)

• Coordinates timing so everything comes together

The manager doesn't cook — they organise.

🔢 The Arithmetic Logic Unit (ALU)

Key Term ALU (Arithmetic Logic Unit)

The component that performs calculations (arithmetic) and logical comparisons. This is where the actual "processing" happens.

The ALU handles two types of operations:

Arithmetic Operations

Mathematical calculations: addition, subtraction, multiplication, division.

Logical Operations

Comparisons and decisions: Is A greater than B? Is this value equal to zero? AND, OR, NOT operations.

🍳 Kitchen Analogy

The ALU is like the cooking station where the actual food gets made. The manager (CU) tells the chef what to cook, but the cooking station is where the work actually happens.

When the manager says "make a calculation," the ALU is what actually does it.

🤔 Why "Logic" Unit?

Programs constantly make decisions: "If the score is greater than 100, show a bonus." The ALU handles these comparisons. Every if statement in code relies on the ALU to check if the condition is true or false.

⚡ Registers

Key Term Registers

Tiny, extremely fast storage locations inside the CPU. They hold the data and instructions the CPU is actively working with at that exact moment.

Registers are the fastest type of memory in the entire computer — even faster than cache. But they're also the smallest — a CPU might only have a handful of registers, each holding just a few bytes.

🍳 Kitchen Analogy

Registers are like the chef's hands. Whatever the chef is working on right now is in their hands — an ingredient being chopped, a pan being stirred. It's the most immediate, instant-access "storage" possible.

You can only hold a few things at once, but access is instant — no reaching for a shelf.

Why So Fast?

Registers are built directly into the CPU chip itself. There's no delay — the CPU can read from or write to a register instantly. RAM is outside the CPU, so it takes longer to access.

💾 Cache: The Bridge to RAM

Key Term Cache

A small amount of high-speed memory inside (or very close to) the CPU. It stores copies of frequently used data and instructions so the CPU doesn't have to wait for slower RAM.

The problem: RAM is useful (it holds all your running programs) but it's slower than the CPU. If the CPU had to wait for RAM every time, it would waste time.

The solution: Cache acts as a middle ground — faster than RAM, but larger than registers.

🍳 Kitchen Analogy

Cache is like the prep station next to the chef. Commonly used ingredients are kept here, already washed and chopped, ready to grab.

RAM is like the pantry — it has much more storage, but you have to walk over to get things. The prep station (cache) keeps the most-used items within arm's reach.

Speed vs Size Trade-off

Memory Type Speed Size Location
Registers Fastest Smallest (bytes) Inside CPU
Cache Very Fast Small (MB) Inside/near CPU
RAM Fast Large (GB) Separate chip
⚠️ Important Relationship

Cache copies data from RAM. When the CPU needs something, it checks cache first. If it's there (a "cache hit"), great — instant access. If not (a "cache miss"), it has to fetch from slower RAM.

🤔 Why Not Make Cache Bigger?

Fast memory is expensive and takes up physical space on the CPU chip. There's always a trade-off: you can have fast OR large, but not both. That's why we have a hierarchy: registers → cache → RAM.

✓ Quick Check

Why does the CPU use cache instead of just accessing RAM directly?

📝 Notes for This Section

Cache

  • Small, high-speed memory inside/near the CPU
  • Faster than RAM, but smaller
  • Stores copies of frequently used data from RAM
  • Reduces wait time for the CPU

Memory Hierarchy (fastest to slowest)

  • Registers → Cache → RAM
  • Trade-off: faster = smaller, larger = slower
