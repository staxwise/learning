Question 1: Merge Sort [4 marks]

🔗 Prior Knowledge: Sorting Algorithms

Merge sort uses divide and conquer: split the list in half repeatedly until you have single items, then merge them back together in order.

🎯 How to Approach This
  1. SPLIT phase: Keep halving until you have individual items
  2. MERGE phase: Combine pairs in order, working back up
  3. Show every stage — the examiner needs to see your process
  4. 4 marks = typically 4 clear stages shown

⚠️ Common Mistakes

• Forgetting to show the split phase
• Not splitting all the way to single items
• Merging without sorting the pairs
• Jumping straight to the final answer

📝 Merge Sort Remember
  • Divide: Split → Split → Split (until single items)
  • Conquer: Merge in order → Merge → Merge (until one list)
  • Always show both phases for full marks

🔗 Prior Knowledge: Bubble Sort

Bubble sort compares adjacent pairs and swaps if they're in the wrong order. Each pass "bubbles" the largest unsorted item to the end. For strings, we compare alphabetically.

🎯 How to Approach This
  1. Compare first two items — swap if needed
  2. Move to next pair, compare, swap if needed
  3. Continue until end of list = one pass complete
  4. Show the state after each complete pass
  5. Stop when no swaps made (or list is sorted)

Pass 3 makes no swaps → list is sorted

💡 Alphabetical Comparison

Compare the first letter of each word. If same, move to second letter, etc.
"bait" < "crime" because 'b' comes before 'c' in the alphabet.

📝 Bubble Sort Remember

  • Show state after each complete pass
  • Stop when no swaps made in a pass

🔗 Prior Knowledge: Binary Search

Binary search requires a sorted list. Find the middle item, compare to target. If target is higher, discard left half; if lower, discard right half. Repeat until found.

🎯 How to Approach This
  1. Find the middle item (for 9 items: index 4 = "orange")
  2. Compare: Is "zebra" before or after "orange"?
  3. "zebra" > "orange" alphabetically → discard left half
  4. Repeat with remaining items
  5. Show which item you check and which half you discard
✓ Model Answer
Step 1
amb hou kic moo orange ran ten win zeb

Middle = "orange". "zebra" > "orange" → discard left

Step 2
amb hou kic moo ora ran tent win zeb

Middle = "tent". "zebra" > "tent" → discard left

Step 3
amb hou kic moo ora ran ten wind zeb

Middle = "wind". "zebra" > "wind" → discard left

Step 4
amb hou kic moo ora ran ten win zebra ✓

Only "zebra" remains → FOUND!

💡 Finding the Middle

For an odd number of items, middle is exact. For even, round down or up consistently.
9 items → middle is index 4 (5th item). Formula: middle = (low + high) DIV 2

📊 Mark Scheme Breakdown

Continues process correctly1 mark

⚠️ Common Mistakes

• Forgetting the list must be sorted first
• Discarding the wrong half
• Not clearly showing which item is the middle
• Stopping before explicitly stating the item is found

📝 Binary Search Remember
  • List MUST be sorted first
  • Find middle → Compare → Discard half → Repeat
  • Much faster than linear search for large datasets

Question 4: Function with 2D Array [6 marks]

4 Create a function, pilotValid(), that:
  • takes a pilot code as a parameter
  • uses the 2D array journeys to calculate hours flown
  • returns "valid" if ≤9 hours or "warning" if >9 hours
  • uses casting where needed
[6]
[0][1]
[0]"SM720""4.5"
[1]"GC730""3"
[2]"JP670""2"
[3]"GC730""3.5"
[4]"CC200""9"
[5]"RY320""12"
🔗 Prior Knowledge

• Functions: Use parameters, return values
• 2D Arrays: Access with [row, column] indexing
• Casting: Converting strings to numbers (the hours are stored as strings!)

🎯 How to Approach This
  1. Function header: Define function with pilot code parameter
  2. Initialise: Set total hours to 0
  3. Loop: Go through each row in the array
  4. Check: If pilot code matches, add hours (with casting!)
  5. Return: "valid" if ≤9, "warning" if >9
✓ Model Answer (Python)
def pilotValid(pilotCode): totalHours = 0 for i in range(6): if journeys[i][0] == pilotCode: totalHours = totalHours + float(journeys[i][1]) if totalHours <= 9: return "valid" else: return "warning"
💡 Why Casting?

The array stores strings ("4.5", "3", etc.). You can't add strings mathematically!
float() converts the string to a decimal number.
Notice GC730 appears twice (rows 1 and 3) — both flights must be added!

⚠️ Common Mistakes

• Forgetting to cast strings to numbers
• Using int() when decimals are needed (use float())
• Only finding the first match (pilot may have multiple flights)
• Using = instead of == for comparison

📝 Function Writing Checklist
  • ✓ Function header with parameter(s) — def name(param):
  • ✓ Initialise any counters/totals
  • ✓ Loop through data structure — for i in range():
  • ✓ Cast data types where needed — float(), int(), str()
  • ✓ Return appropriate value(s)

Question 7: Test Data Types [4 marks]

7 A program accepts integer values between 1 and 100 (inclusive).
Tick one or more boxes to identify the type(s) of test for each example data. [4]
🔗 Prior Knowledge: Test Data Types

Normal: Typical valid data within expected range
Boundary: At the edges of valid/invalid (1, 100, 0, 101)
Erroneous: Invalid data the program should reject

✓ Answers
Test DataNormalBoundaryErroneous
27✓
"Hello"✓
105✓
100✓
💡 Boundary vs Erroneous

100 is boundary because it's the edge of valid range (1-100 inclusive).
105 is erroneous because it's outside the valid range entirely.
"Hello" is erroneous — wrong data type entirely.

📝 Test Data Types
  • Normal: Valid, typical, middle-of-range values
  • Boundary: Edge values (min, max, just inside/outside)
  • Erroneous: Invalid data (wrong type, out of range)

Question 9: Algorithm Writing [6 marks]

9 Write an algorithm to play a game with the following rules:
  • the player is asked 3 addition questions
  • each question asks the player to add two random whole numbers between 1 and 10 inclusive
  • if the player gets the correct answer, 1 is added to their score
  • at the end of the game their score is displayed
[6]
🔗 Prior Knowledge

• Iteration: FOR loop for fixed number of repeats
• Random numbers: import random then random.randint(1, 10)
• Selection: IF to check answer
• Variables: Counter for score

🎯 How to Approach This
  1. Import: import random
  2. Initialise: Set score = 0
  3. Loop 3 times: for i in range(3):
  4. Generate: Two random numbers (1-10)
  5. Calculate: The correct answer
  6. Ask: Display question, get user answer
  7. Check: IF answer correct THEN add 1 to score
  8. Output: Display final score
✓ Model Answer (Python)
import random score = 0 for i in range(3): num1 = random.randint(1, 10) num2 = random.randint(1, 10) correctAnswer = num1 + num2 print("What is " + str(num1) + " + " + str(num2) + "?") userAnswer = int(input("Your answer: ")) if userAnswer == correctAnswer: score = score + 1 print("Your final score is: " + str(score))
💡 Alternative Approaches

• Score increment: score = score + 1 or score += 1
• Print: Can use f-strings: print(f"Your score is: {score}")
• Remember to import random at the top!

📊 Mark Scheme Breakdown

Initialise score to 01 mark
Loop 3 times1 mark
Generate two random numbers (1-10)1 mark
Ask question and get input1 mark
Check answer and update score1 mark
Output final score1 mark
⚠️ Common Mistakes

• Forgetting to initialise score BEFORE the loop
• Outputting score INSIDE the loop (should be after)
• Using wrong random range (must be 1-10 inclusive)
• Comparing strings to integers without casting

📝 Algorithm Writing Checklist
  • ✓ Read ALL bullet points — each is usually worth a mark
  • ✓ Initialise counters/totals BEFORE loops
  • ✓ Use FOR when you know how many times to repeat
  • ✓ Output should happen AFTER the loop ends
