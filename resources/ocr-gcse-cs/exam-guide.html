📑 The Two Papers

Your GCSE Computer Science qualification is split equally between two papers. You must sit both.

Component 01
Computer Systems
80 marks • 1 hour 30 mins • 50%

Theory paper covering hardware, software, networks, security, and ethics.
Component 02
Computational Thinking, Algorithms & Programming
80 marks • 1 hour 30 mins • 50%

Problem-solving, algorithms, and programming skills.

marks each
🔗 Synopticity

Questions can draw on knowledge from both papers. For example, a programming question in Paper 2 might involve concepts from Paper 1 (like how the CPU works). Everything connects!

📝 Key Points
  • Two papers, each worth 50% (80 marks)
  • 1 hour 30 minutes for each paper
  • Paper 1 = Theory, Paper 2 = Programming & Problem-solving
  • Questions can link knowledge from across the whole course
Learning Objective

Understand what AO1, AO2, and AO3 mean and how they affect your answers.

🎯 Assessment Objectives

Every exam question tests one or more Assessment Objectives (AOs). Understanding these helps you know what the examiner is looking for.

AO1: Knowledge & Understanding 30%

Show you know the facts, concepts, and terminology.

AO2: Application 40%

Apply your knowledge to scenarios and problems.

AO3: Analysis & Problem-Solving 30%

Analyse problems, make judgements, design and write programs.

Match the AO: A question asks you to "State two advantages of using an SSD over an HDD." Which AO is this primarily testing?

💡 What This Means For You

40% of marks are AO2 — that's application. You'll often need to apply your knowledge to a specific context or scenario, not just state facts. Read questions carefully!

📝 Key Points
  • AO1 (30%): Know the facts — definitions, terminology, concepts
  • AO2 (40%): Apply knowledge — use it in scenarios/contexts
  • AO3 (30%): Analyse & solve — design programs, make judgements
  • Most marks require more than just memorisation!
Learning Objective

Understand what Component 01 contains and how the 8-mark extended response works.

📘 Component 01: Computer Systems

This is your theory paper. It tests your understanding of how computers work, from hardware to ethics.

What's Covered

  • Systems architecture (CPU, memory)
  • Memory and storage
  • Networks, connections, and protocols
  • Network security
  • Systems software
  • Ethical, legal, cultural, and environmental issues
Special Question 8-Mark Extended Response

Component 01 always has one 8-mark question that tests your ability to construct a detailed, logical argument. These use a "levels of response" mark scheme.

Example 8-Mark Question
"People often want to buy the most up-to-date smartphones, even though the smartphone they own still works.

Discuss the impact of people wanting to upgrade to the latest smartphone.

In your answer you might consider the impact on: smartphone users, cultural issues, ethical issues, environmental issues."

Think about it: For an 8-mark "Discuss" question, which approach would get you more marks?

💡 Extended Response Tips

For 8-mark questions: Plan before you write. Cover multiple aspects (the bullet points are hints!). Develop your points with explanation. Use a logical structure. A conclusion can help show balanced judgement.

📝 Key Points
  • Component 01 is the theory paper (hardware, software, networks, security, ethics)
  • Contains ONE 8-mark extended response question
  • 8-mark questions test your ability to discuss and evaluate, not just list
  • Use the bullet points in the question as a guide for structure

Section A covers computational thinking, algorithms, and general programming concepts.

Pseudocode

📖 What is ERL?

The Exam Reference Language is OCR's standardised pseudocode. You don't need to memorise it or write in it — but you do need to be able to read and understand it when it appears in questions.

Check your understanding: A Section A question asks you to "write an algorithm to find the largest number in a list." Can you respond with a flowchart?

💡 Section A also has a 6-mark question

Section A always includes one larger 6-mark question. This is still marked using a points-based scheme (not levels of response like the 8-marker in Paper 1).

📝 Key Points
  • Section A = 50 marks of computational thinking and algorithms
  • Flexible responses: pseudocode, flowcharts, bullet points, ERL, or actual code
  • ERL: You need to READ it, not memorise it
  • Contains one 6-mark question (points-based marking)
Learning Objective

Understand Component 02 Section B and the four question types: Design, Write, Test, and Refine.

📗 Component 02: Section B (30 marks)

Section B tests your actual programming skills. Questions build on each other but aren't dependent — you can still answer later questions even if you struggled with earlier ones.

⚠️ Less Flexibility Here!

Section B has stricter rules about how you respond. Some questions require you to use ERL or a high-level programming language — flowcharts and bullet points won't be accepted for those.

The Four Question Types

Section B questions fall into four categories:

Type What It Tests Allowed Responses
Design Creating ideas for algorithms ✓ Pseudocode, flowchart, bullet points, code
Write Writing actual program code ✓ ERL or high-level language ONLY
Test Testing programs work correctly ✓ Trace tables, test data, test cases
Refine Fixing/improving existing code ✓ ERL or high-level language ONLY

Quick check: A question says "Write a program to calculate the average of numbers in an array." Can you answer using a flowchart?

💡 Look For This Instruction

Questions that require code will clearly state: "You must use either: OCR Exam Reference Language, or a high-level programming language that you have studied."

📝 Key Points
  • Section B = 30 marks testing practical programming
  • Design: Flexible (pseudocode, flowchart, etc.)
  • Write: Must use ERL or actual code
  • Test: Trace tables and test data
  • Refine: Must use ERL or actual code
Learning Objective

Understand how programming answers are marked and what "precision" means.

⌨️ Writing Code in Exams

Between 10-15% of Component 02 (that's 8-12 marks) specifically tests your ability to write algorithms with precision.

🎯 What Does "Precision" Mean?

OCR tests that you can use correct programming syntax — the common elements that appear across all languages (variables, loops, conditions, etc.).

What's Penalised vs What's Not

✓ Minor Errors Usually NOT Penalised

A missing colon or semicolon won't lose marks if the examiner can still understand what your code is meant to do.

✗ These WILL Be Penalised

Errors that show misunderstanding, like writing 7 = x instead of x = 7 (assignment the wrong way round).

Which of these would likely lose marks?

💡 Best Practice

Write in the language you've used in class — you'll make fewer mistakes. Stronger candidates typically use their classroom language rather than trying to use ERL.

📝 Key Points
  • 8-12 marks test precise algorithm writing
  • Minor syntax errors (colons, semicolons) usually not penalised
  • Logic errors and wrong operations ARE penalised
  • Use the language you know best from class
Learning Objective

Recognise the three main question types and understand how each is marked.

❓ Question Types

Questions fall into three broad categories based on how many marks they're worth:

1-3 marks Short Answer
Quick recall or simple application. May be tick boxes, fill-in-the-blank, or brief written answers.
3-6 marks Medium Response
More detailed answers or algorithm writing. Points-based marking — each correct point earns marks.
8 marks Extended Response
Only in Paper 1. Requires discussion/evaluation. Marked using levels (quality of argument, not just number of points).

Short Answer Examples

These are designed to be accessible and test knowledge efficiently:

  • Tick box: "Tick one box to identify which is a comparison operator"
  • Fill in the blank: "The process of _____ breaks down a problem into smaller parts"
  • State/Identify: "Identify three devices that contain embedded systems"

Quick check: A question worth 6 marks in Component 02 Section A would be marked using...

📝 Key Points
  • Short (1-3): Quick answers, tick boxes, fill blanks
  • Medium (3-6): More detail, points-based marking
  • Extended (8): Paper 1 only, levels of response
  • The 6-mark in Section A is points-based, NOT levels
Learning Objective

Understand key command words and what type of response each one requires.

📢 Command Words

Command words tell you exactly what kind of response is expected. Misreading them is one of the most common ways to lose marks!

State / Identify / Name
Brief answer — no explanation needed
Describe
Say what something is or how it works
Explain
Give reasons WHY — more detail than describe
Discuss
Consider different aspects/viewpoints
Compare
Show similarities AND differences
Write / Complete
Produce code/algorithm — check if code required

Spot the difference: How would your answer differ between these two questions?

A: "State one advantage of using an SSD" [1 mark]

B: "Explain one advantage of using an SSD" [2 marks]

💡 Example Answers

State (1 mark): "Faster read/write speeds"

Explain (2 marks): "Faster read/write speeds [1] because SSDs have no moving parts so data can be accessed instantly [1]"

📝 Key Points
  • State/Identify = brief, no explanation
  • Describe = what/how
  • Explain = why (give reasons)
  • Discuss = multiple viewpoints/aspects
  • More marks usually = more detail expected
Learning Objective

Understand when and why showing your working matters.

✏️ Show Your Working

Some questions explicitly ask you to show your working. This is especially common for:

  • Binary, denary, and hexadecimal conversions
  • Data representation calculations
  • Any mathematical computation
Important Partial Credit

If you show your working, you can still earn marks even if your final answer is wrong! The examiner can see where you went wrong and credit the correct steps.

Example: Converting 156 to Binary
Good working:
156 - 128 = 28 ✓
28 - 64 = can't
28 - 32 = can't
28 - 16 = 12 ✓
12 - 8 = 4 ✓
4 - 4 = 0 ✓
Answer: 10011100

Even if you made an arithmetic error, correct method = marks!

True or false: If a question doesn't ask for working, you should never show it.

📝 Key Points
  • Show working when asked — especially for conversions
  • Working can earn partial credit even if final answer is wrong
  • Good practice to show working even when not required
  • Working helps you avoid mistakes too!
